# Accessibility Compliance Checklist for Merchant Onboarding

This checklist is based on the accessibility compliance review against WCAG criteria for the merchant onboarding feature specifications.

## General Portal Requirements
- Ensure the portal is fully operable with keyboard-only navigation.
- Provide screen reader support with proper semantic HTML and ARIA roles.
- Use clear and simple language in all instructions and UI text.
- Maintain sufficient color contrast ratios for text and UI components.
- Ensure text is scalable and respects user font size preferences.
- Provide visible focus indicators for all interactive elements.
- Avoid using color as the sole means of conveying information.
- Provide text alternatives for all non-text content (e.g., images, icons).

## Guided Submission Process
- Implement accessible tooltips that are keyboard and screen reader accessible.
- Use clear, concise instructions and examples.
- Ensure error messages are programmatically associated with the relevant form fields.
- Provide suggestions and guidance for correcting errors.

## Progress Indicators
- Use ARIA live regions or text alternatives to communicate progress updates to screen readers.
- Ensure progress is not conveyed by color alone; include text or icons.

## Email Notifications
- Structure emails with proper heading levels and semantic markup.
- Use meaningful link text and ensure links are keyboard accessible.
- Ensure sufficient color contrast in email text and backgrounds.
- Allow text scalability within email clients.

## Integration and Third-Party Services
- Validate that third-party services used (e.g., KYB verification) meet accessibility standards.
- Ensure seamless and accessible integration without disrupting user experience.

## Testing and Validation
- Conduct user testing with keyboard-only users and screen reader users.
- Use automated accessibility testing tools to identify issues.
- Perform manual testing for color contrast and focus management.

## Communication and Support
- Provide accessible means for users to inquire about manual review status or get help.
- Include clear instructions and FAQs that are accessible.

---

Follow this checklist to help ensure the merchant onboarding portal and related communications are accessible and compliant with WCAG standards.
