# Merchant Onboarding User Experience Improvement Issues

## Identified Issues

1. **Submission Portal Experience**  
   - **User-Friendly Interface**: The submission portal must be intuitive and easy to navigate to prevent user frustration. This can include simplifying forms and providing clear instructions.
   - **Guidance and Help Resources**: Implement contextual help or tooltips to assist users in understanding document requirements during submission.

2. **Automated Verification Process**  
   - **Transparency of Process**: Clearly inform users about the automated verification process and the necessary documents for successful verification.
   - **Feedback on Verification Status**: Provide real-time feedback on the verification status to keep users informed if additional information is needed.

3. **Manual Review Queue for Edge Cases**  
   - **Clear Communication of Delays**: Notify users promptly about any delays in manual reviews and the reasons for these delays to manage expectations.
   - **Streamlined Manual Review Process**: Ensure efficiency in the manual review process and provide users with a way to inquire about their application status without excessive waiting.

4. **Email Notifications**  
   - **Timely Notifications**: Ensure prompt email notifications regarding approval or rejection after the verification process.
   - **Clarity in Communication**: Notifications should clearly explain outcomes and provide next steps or resources for users who are rejected.

5. **Overall User Journey**  
   - **Mapping User Journey**: Analyze the entire onboarding journey to identify pain points such as long wait times and unclear instructions.
   - **User Feedback Mechanism**: Implement a feedback mechanism for users to share their onboarding experience, which can help in identifying areas for improvement.

## Actionable Improvement Issues

1. Redesign the submission portal to enhance usability, incorporating user testing feedback.
2. Develop a detailed guide on the automated verification process and create a status tracking feature for users.
3. Implement a notification system for manual review delays and streamline communication processes.
4. Audit email notification templates for clarity and ensure timely delivery of information to users.
5. Conduct user journey mapping sessions and establish a user feedback loop for continuous improvement.
